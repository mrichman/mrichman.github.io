--- layout: post title: Interacting with SOAP based web services from Cocoa categories: [articles] tags: - soap mac cocoa ruby webservices xml rest iphone xcode status: publish type: post published: true meta: _edit_last: '1' dsq_thread_id: '92599892' --- SOAP based web services and Cocoa have never been good friends, and although REST based webservices are fortunately today's standard there are still tons of SOAP-based ones out there. What has always been missing in Cocoa is high level support for interacting with SOAP-based webservices. Here I describe one way in which you can build such frameworks yourself, starting by analysing the webservice calls in detail in this part 1 and generating the necessary Cocoa classes for use in a Mac or iPhone application next time. http://www.macresearch.org/interacting-soap-based-web-services-cocoa-part-1
